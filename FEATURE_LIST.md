# ChickenNoteLM Feature List

> 更新时间：2026-03-17  
> 部署路径：`/var/www/chickennoteLM`

## 1) 账号与访问控制

- **用户登录**：用户名登录，服务端 Session 管理（`/api/auth/login`）
- **登录态校验**：页面初始化时调用 `/api/auth/me`，未登录强制停留在登录页
- **退出登录**：清除 Session 并刷新页面（`/api/auth/logout`）
- **用户数据隔离**：每个用户拥有独立的数据目录
  - `notefile/<user_id>/` - 笔记数据（*.md + *.json）
  - `chatdata/<user_id>/` - 聊天记录
  - `eventdata/<user_id>/` - 事件数据
  - `sync_state/<user_id>/` - 同步状态
  - `uploads/<user_id>/<note_id>/` - 图片上传

## 2) 笔记编辑与管理

- **笔记生命周期**：新建、读取、更新、删除（CRUD）
- **笔记列表展示**：标题、更新时间、内容摘要、多选复选框
- **笔记切换**：点击列表项加载正文到编辑器
- **自动保存**：输入防抖（debounce），自动同步到 IndexedDB 和服务器
- **手动保存**：Ctrl+S 或保存按钮触发的即时保存
- **重命名笔记**：支持修改笔记标题，自动更新文件名
- **批量操作**：多选、全选、批量删除
- **编辑模式**：Markdown 源码编辑 / 预览模式切换
- **AI 格式化**：将普通文本转换为规范 Markdown 格式

## 3) 数据存储与同步

### 本地存储（前端）
- **IndexedDB**：主离线存储，存储笔记、事件、聊天记录
- **localStorage**：设置存储、临时状态

### 服务端存储
- **笔记落盘**：Markdown 内容（*.md）+ 元数据（*.json）
- **笔记索引**：`notes_index.json` 维护笔记元数据索引，支持快速加载
- **事件落盘**：`events.json` 存储所有事件数据
- **聊天落盘**：每个会话独立的 JSON 文件

### 增量同步机制
- **同步状态查询**：`/api/sync/status` - 获取服务器笔记修改时间戳
- **增量拉取**：`/api/sync/pull` - 从服务器拉取更新的笔记
- **增量推送**：`/api/sync/push` - 推送本地变更到服务器
- **冲突解决**：`/api/sync/resolve` - 处理服务器与客户端版本冲突
- **设备标识**：支持多设备通过 `device_id` 区分同步状态

### 网络感知
- **在线状态检测**：监听 `online/offline` 事件
- **后台同步**：Service Worker 后台同步触发
- **离线提示**：网络断开时显示提示，恢复后自动同步

## 4) AI 能力

### AI 对话
- **上下文对话**：基于当前选中笔记内容进行问答
- **多会话管理**：新建、读取、保存、列表展示会话
- **系统提示词**：可自定义对话系统提示词

### 事件提取
- **单篇提取**：从当前笔记提取结构化事件
- **全局提取**：从所有笔记批量提取事件
- **智能解析**：自动识别时间、任务、截止日期等信息

### AI 格式化
- **文本转 Markdown**：一键将普通文本转换为规范 Markdown
- **结果回写**：AI 转换结果可直接覆盖编辑器内容
- **撤销支持**：保留格式化历史，支持撤销操作

### OpenClaw 代理
- **健康检查**：`/api/openclaw/health`
- **聊天代理**：`/api/openclaw/chat` - 代理到 OpenClaw API
- **TUI WebSocket**：`/api/openclaw/tui` - 在页面内运行 openclaw tui

## 5) 可配置项（Settings）

- **API Key**：OpenAI 或兼容服务的密钥
- **Base URL**：API 基础地址
- **模型名称**：如 `gpt-3.5-turbo`、`gpt-4` 等
- **系统提示词**：
  - 对话系统提示词
  - 事件提取系统提示词
  - 文本转 Markdown 提示词（可自定义）
- **设置持久化**：本地存储，支持历史配置兼容迁移

## 6) 事件模块

- **事件提取**：单篇提取、全局提取
- **事件列表**：卡片式展示，支持展开/折叠
- **事件编辑**：标题、上下文、标签编辑
- **标签管理**：新增标签、删除标签、按标签过滤
- **事件排序**：创建时间、更新时间，支持升序/降序
- **数据持久化**：事件数据同步到服务器

## 7) 图片上传

- **粘贴上传**：支持从剪贴板粘贴图片
- **拖拽上传**：支持拖拽图片到编辑器
- **自动关联**：图片自动关联到当前笔记
- **分目录存储**：按用户 ID 和笔记 ID 分目录存储
- **URL 生成**：生成可访问的图片 URL（`/uploads/{user_id}/{note_id}/{filename}`）
- **格式支持**：PNG、JPG、GIF、WebP、SVG

## 8) 导入导出

- **导入 JSON**：支持导入笔记和事件的 JSON 数据
- **导出笔记 JSON**：导出所有笔记为 JSON 格式
- **导出事件 JSON**：导出所有事件为 JSON 格式
- **导出 Markdown**：导出所有笔记为单个 Markdown 文件

## 9) 前端离线与缓存

- **Service Worker 注册**：`sw.js` 提供离线支持
- **静态资源缓存**：缓存核心 JS/CSS/HTML 文件
- **缓存策略**：
  - API 请求：网络优先
  - 静态资源：缓存优先 + 后台更新
- **缓存版本管理**：当前缓存名 `chickennoteLM-v9`
- **后台同步**：`sync` 事件触发笔记同步

## 10) 后端能力（Flask）

### 认证 API
- `GET /api/auth/me` - 获取当前登录用户
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/logout` - 用户登出

### 笔记 API
- `GET /api/notes` - 获取当前用户所有笔记
- `POST /api/sync/notes-events` - 全量同步笔记和事件

### 事件 API
- `GET /api/events` - 获取事件列表

### 聊天 API
- `GET /api/chats` - 列出所有聊天会话
- `GET /api/chats/<id>` - 获取单个会话
- `POST /api/chats` - 创建新会话
- `POST /api/chats/<id>` - 保存会话

### 同步 API
- `GET /api/sync/status` - 获取服务器同步状态
- `POST /api/sync/pull` - 从服务器拉取变更
- `POST /api/sync/push` - 推送变更到服务器
- `POST /api/sync/resolve` - 解决同步冲突

### 上传 API
- `POST /api/uploads/image` - 上传图片（需登录，支持 multipart/form-data）

### OpenClaw 代理
- `GET /api/openclaw/health` - 健康检查
- `POST /api/openclaw/chat` - 聊天代理
- `WS /api/openclaw/tui` - TUI WebSocket

## 11) 已知约束

- **无痕模式**：浏览器本地缓存（IndexedDB/localStorage/cookie）不稳定，可能导致数据不可见
- **模型依赖**：AI 能力依赖外部 API 可用性与 Key 配置
- **轻量登录**：当前为轻量用户名模式，适合内测，不等同于生产级认证体系
- **并发同步**：多设备同时编辑同一笔记可能产生冲突，需手动解决
